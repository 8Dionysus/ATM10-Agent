# SESSION_2026-02-27

Краткий конспект сессии от **2026-02-27** (`M7.1` + `M7.2` + `M8.0` + `M8.1` + `M7.post` pass).

## Что сделано

* Закрыт `M7.1`: добавлен HTTP transport поверх существующего `gateway_v1` контракта без drift бизнес-логики:
  * `scripts/gateway_v1_http_service.py`
  * `GET /healthz`
  * `POST /v1/gateway` (thin-wrapper над `run_gateway_request`)
* Закрыт `M7.2`: выполнен hardening HTTP слоя без смены body schema:
  * policy defaults (`Balanced`): `max_request_body_bytes=262144`, `max_json_depth=8`, `max_string_length=8192`, `max_array_items=256`, `max_object_keys=256`, `operation_timeout_sec=15.0`
  * pre-validation request body (size + JSON limits)
  * timeout policy: `operation_timeout -> HTTP 504`
  * sanitized internal error envelope: `internal_error_sanitized`, traceback в локальном `gateway_http_errors.jsonl`
* Расширен HTTP status mapping:
  * `status=ok` -> `200`
  * `invalid_request` -> `400`
  * `payload_too_large|payload_limit_exceeded` -> `413`
  * `operation_timeout` -> `504`
  * `operation_failed|gateway_dispatch_failed|internal_error_sanitized` -> `500`
* Обновлен HTTP smoke summary contract:
  * добавлены поля `expected_http_status`, `http_status`, `ok`, `failed_requests_count`
* Добавлен parity matrix regression:
  * `tests/test_gateway_v1_contract_parity_matrix.py` (`CLI vs HTTP` для 4 операций)
* CI smoke summary для HTTP расширен колонкой `failed_requests_count`.
* Синхронизированы source-of-truth docs: `RUNBOOK`, `DECISIONS`, `TODO`, `PLANS`.
* Закрыт `M8.0`: добавлен decision-complete IA spec для Streamlit panel v0:
  * `docs/STREAMLIT_IA_V0.md`
  * обязательные зоны, data contracts, error states, safe guardrails, `M8.1` handoff contract
* Добавлен doc-contract regression test:
  * `tests/test_streamlit_ia_doc.py`
* Закрыт `M8.1`: реализован Streamlit operator panel v0 и no-crash smoke gate:
  * `scripts/streamlit_operator_panel.py` (4 зоны UI из IA, canonical sources, gateway health, safe actions whitelist)
  * `scripts/streamlit_operator_panel_smoke.py` (`streamlit_smoke_summary_v1`, startup timeout/terminate policy)
* Добавлены regression tests для M8.1:
  * `tests/test_streamlit_operator_panel.py`
  * `tests/test_streamlit_operator_panel_smoke.py`
* CI smoke workflow расширен Streamlit шагом:
  * `runs/ci-smoke-streamlit/streamlit_smoke_summary.json`
  * summary section `Streamlit Smoke Summary`
  * artifact upload path для Streamlit summary
* Runtime dependencies обновлены:
  * в `requirements.txt` добавлен `streamlit`.
* Закрыт `M7.post`: добавлен SLA/observability baseline поверх gateway smoke summaries:
  * `scripts/gateway_v1_smoke.py` и `scripts/gateway_v1_http_smoke.py` расширены полями observability:
    * scenario-level `started_at_utc`, `finished_at_utc`, `duration_ms`
    * per-request `latency_ms`
    * aggregate `latency_p50_ms`, `latency_p95_ms`, `latency_max_ms`
    * `failed_requests_count`, `error_buckets`
  * добавлен `scripts/check_gateway_sla.py` c контрактом `gateway_sla_summary_v1`
  * добавлен policy-aware SLA режим:
    * `signal_only` -> не фейлит на breach
    * `fail_on_breach` -> exit `2` при breach
  * CI расширен SLA шагом, SLA summary section и artifact:
    * `runs/ci-smoke-gateway-sla/gateway_sla_summary.json`
