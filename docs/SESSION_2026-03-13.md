## Update (same date): G2 single-cycle operator helper

### What was added

* Added new local helper `scripts/run_gateway_sla_operating_cycle.py` with contract `gateway_sla_operating_cycle_v1`.
* The helper performs one full end-to-end `G2` operating cycle:
  * first reads latest `readiness/governance/progress/transition/remediation/integrity/cadence`
  * determines whether required summaries are fresh for the current UTC day
  * if the latest snapshot is already fresh, reuses it without a new `manual_nightly`
  * if the latest snapshot is stale/missing/invalid, runs fixed-order fallback `manual_nightly -> cadence -> remediation -> integrity`
* The helper writes:
  * machine-readable summary `runs/nightly-gateway-sla-operating-cycle/operating_cycle_summary.json`
  * short human-facing brief `runs/nightly-gateway-sla-operating-cycle/triage_brief.md`
* Added regression tests for:
  * reusing a fresh manual-backed snapshot

* `docs/RUNBOOK.md` was updated: the helper is documented as the preferred local operator entrypoint for a single `G2` cycle.
