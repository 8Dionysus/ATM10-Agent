# SESSION_2026-02-24

Краткий конспект сессии от **2026-02-24** (audit-fixes pass).

## Что сделано

* Закрыт security-риск в `scripts/voice_runtime_service.py`:
  * `out_wav_path` ограничен до безопасного имени файла (без absolute/path traversal).
  * TTS output всегда записывается только в server run artifact: `runs/<timestamp>-voice-service/tts_outputs/`.
* Для совместимости обновлен `scripts/voice_runtime_client.py`:
  * клиент отправляет в `out_wav_path` только basename (без absolute path), чтобы не ломать `tts/tts-stream` на новом серверном контракте.
* Убран traceback leakage в HTTP/NDJSON ответах `voice_runtime_service`:
  * клиент получает только `error_code=internal_error` + короткий `error`.
  * traceback сохраняется локально в `service_errors.jsonl`.
* Исправлен operational дефект в `scripts/retrieve_demo.py`:
  * при backend error теперь всегда пишется `run.json` (`status=error`, `error_code=retrieval_backend_error`).
* Исправлен direct script-run path для `scripts/export_qwen3_tts_openvino.py`:
  * добавлен fallback import для `python scripts/export_qwen3_tts_openvino.py ...`.
* Исправлен flaky-case в `scripts/discover_instance.py`:
  * создание run-dir стало collision-safe при одинаковом timestamp (suffix-loop).
* Закрыт supply-chain риск в `scripts/tts_runtime_service.py` (Silero path):
  * remote `torch.hub` источник заблокирован по умолчанию;
  * для remote требуется explicit opt-in (`SILERO_ALLOW_REMOTE_HUB=true`) и pinned revision (`SILERO_REPO_REF` или `owner/repo:ref`);
  * local source path остается разрешенным без opt-in.
