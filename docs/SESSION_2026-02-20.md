## Voice latency (in-game SLA check)

Service mode (`voice_runtime_service + voice_runtime_client`):

* ASR cold: `9.141s`; warm p50/p95: `0.775s / 0.847s`
* TTS cold: `24.618s`; warm p50/p95: `9.688s / 14.747s`

One-shot mode (`asr_demo` / `tts_demo`):

* ASR p50/p95: `13.352s / 14.016s`
* TTS p50/p95: `22.445s / 24.169s`

Вывод: service mode существенно быстрее one-shot, но in-game SLA `<=2s` на текущем CPU runtime выполняется только для ASR; bottleneck — `Qwen3-TTS`.
